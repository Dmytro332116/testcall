## 🔄 Деталізований процес дзвінка

### Фаза 1: Прийом дзвінка

```
1. Користувач дзвонить на +380(73)383 0069
2. ВХІДНИЙ INVITE повідомлення на SIP сервер
3. SIP сервер маршрутизує на Node.js агента
4. Агент отримує 'newRTCSession' подію
5. Агент автоматично відповідає (Answer)
6. Встановлюється WebRTC з'єднання
7. Аудіо потік активний
```

**Коди статусів:**
```
100 Trying        → Шукаємо адресата
180 Ringing       → Дзвінок йде
183 Session Progress → З'єднання в процесі
200 OK           → Дзвінок прийнято
```

### Фаза 2: Привіт від AI

```
1. Agenta генерує привіт: "Привіт! Це ШІ асистент..."
2. Текст відправляється на ElevenLabs API
3. ElevenLabs конвертує текст в аудіо (WAV 8kHz)
4. Аудіо буферизується в пам'яті
5. PCM аудіо передається в WebRTC потік
6. Користувач чує голос агента
```

### Фаза 3: Розпізнавання мови користувача

```
1. WebRTC обчислює аудіо зі сторони користувача
2. Audio chunks накопичуються в буфер
3. Коли користувач припиняє говорити:
   a) Виявляється тиша (silence detection)
   b) Аудіо буфер відправляється на Whisper
4. Whisper розпізнає українську мову
5. Повертається текст: "Це було чудово!"
```

### Фаза 4: Обробка AI

```
1. Текст від користувача передається GPT-4
2. Разом з історією розмови:
   - System Prompt (інструкції для AI)
   - Previous messages (контекст)
   - Current user message (поточне повідомлення)
3. GPT-4 генерує відповідь українською
4. Система пам'яті зберігає в conversationHistory
5. Повертається текст відповіді
```

### Фаза 5: TTS та передача

```
1. Текст відповіді:    "Спасибо! Як я можу допомогти?"
2. Відправляється на ElevenLabs
3. ElevenLabs стримує аудіо chunks
4. Кожен chunk одразу передається в дзвінок
5. Виключається затримка (low-latency streaming)
6. Користувач чує відповідь природно
```

### Фаза 6: Завершення дзвінка

```
1. Після 5-10 турів розмови агент каже:
   "Дякую за розмову! До побачення!"
2. Отправляет BYE повідомлення
3. Закривається WebRTC з'єднання
4. Записуються логи розмови (опціонально)
5. Скидається conversationHistory
6. Система готова до наступного дзвінка
```

## 📈 Продуктивність

### Затримки (Latency)

```
╔══════════════════════════════════════════╗
║         ЗАТРИМКИ НА ЕТАПАХ              ║
╠══════════════════════════════════════════╣
║ 1. SIP з'єднання:           ~100-200ms   ║
║ 2. Audio Capture:           ~20-50ms     ║
║ 3. Whisper STT:             ~500-1500ms  ║
║ 4. GPT-4 Processing:        ~1000-3000ms ║
║ 5. ElevenLabs TTS:          ~500-1500ms  ║
║ 6. Audio Playback:          ~20-50ms     ║
║                                         ║
║ ВСЬОГО НА ХОДА:             ~2600-6300ms ║
║ (2.6 - 6.3 секунди)                     ║
╚══════════════════════════════════════════╝
```

**Оптимізація:**
- Streaming (не чекаємо повного аудіо перед відтворенням)
- Паралельна обробка (якщо можливо)
- Кешування відповідей (для частих питань)

### Пропускна спроможність

```
Один агент може обробляти:
- ~10-20 одночасних дзвінків (залежить від серверу)
- Тривалість дзвінка: ~5-15 хвилин середня
- Перерва між дзвінками: ~1-2 секунди
```

Масштабування:
```
Для 1000 дзвінків на день:
- ~50 паралельних дзвінків
- Потрібно ~5 серверів (10 дзвінків кожен)
- Або використовуємо контейнери (Docker Swarm/Kubernetes)
```

## 💾 База даних (Опціонально)

### Структура для зберігання розмов

```sql
CREATE TABLE conversations (
  id UUID PRIMARY KEY,
  caller_number VARCHAR(20),
  agent_name VARCHAR(100),
  duration_seconds INT,
  status ENUM('completed', 'failed', 'timeout'),
  started_at TIMESTAMP,
  ended_at TIMESTAMP,
  transcript TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE messages (
  id UUID PRIMARY KEY,
  conversation_id UUID FOREIGN KEY,
  role ENUM('user', 'assistant'),
  content TEXT,
  timestamp TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE call_logs (
  id UUID PRIMARY KEY,
  conversation_id UUID FOREIGN KEY,
  event_type VARCHAR(50),
  details JSON,
  timestamp TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Інтеграція з БД

```javascript
// Приклад збереження розмови
async function saveConversation(callData) {
  const { callerId, agentName, duration, transcript } = callData;
  
  await db.query(
    'INSERT INTO conversations (caller_number, agent_name, duration_seconds, transcript) VALUES (?, ?, ?, ?)',
    [callerId, agentName, duration, transcript]
  );
}
```
