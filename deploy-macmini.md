# SKILL: Deploy AI Voice Agent on Mac Mini (via Docker)

## Мета
Твоя задача — розгорнути систему ШІ-телефонії (Node.js + Asterisk + ElevenLabs) на цьому Mac Mini. Проект повністю контейнеризований (Docker), тому твоя задача зводиться до клонування, налаштування змінних середовища та запуску контейнерів.

## Кроки для виконання (Step-by-Step):

### Крок 1: Клонування репозиторію
1. Перейди в робочу директорію, де зазвичай лежать проекти (наприклад, `~/Projects` або `~/Desktop`).
2. Виконай команду: `git clone https://github.com/Dmytro332116/testcall.git`
3. Перейди в папку проекту: `cd testcall`

### Крок 2: Налаштування змінних середовища (.env)
1. У папці `testcall` знайди файл `.env.example`.
2. Скопіюй його у новий файл `.env`: `cp .env.example .env`
3. Звернись до користувача з проханням надати ключі:
   - "Будь ласка, надайте ваш `ELEVENLABS_API_KEY`, щоб я міг додати його у файл .env"
4. Щойно користувач дасть ключ, відкрий файл `.env` і заміни значення `ELEVENLABS_API_KEY` на отримане.
5. Переконайся, що `ELEVENLABS_AGENT_ID=agent_3701kp6je3rsf8nt65rswhzh69cj` вказано правильно.

### Крок 3: Перевірка Docker
1. Виконай команду `docker --version` та `docker-compose --version`, щоб переконатися, що Docker запущений на цьому Mac Mini.
2. Якщо Docker не запущений, ввічливо попроси користувача запустити Docker Desktop (або OrbStack).

### Крок 4: Запуск системи
1. Виконай збірку та запуск контейнерів у фоновому режимі:
   `docker-compose up -d --build`
2. Переконайся, що обидва контейнери (`ai-voice-agent` та `asterisk-pbx`) працюють, виконавши `docker ps`.
3. Перевір логи Node.js сервера на відсутність помилок:
   `docker logs ai-voice-agent --tail 20`

### Крок 5: Тестовий дзвінок
1. Повідом користувачу, що система успішно запущена.
2. Запитай: "На який номер зробити тестовий дзвінок? (формат: 380XXXXXXXXX)"
3. Коли користувач дасть номер, виконай команду:
   `curl -X POST http://127.0.0.1:3000/call -d '{"number":"ЙОГО_НОМЕР"}' -H 'Content-Type: application/json'`
4. Скажи користувачу приймати дзвінок!

## Правила
- Не намагайся встановлювати Asterisk через `brew install asterisk`. Використовуй виключно `docker-compose`.
- Якщо виникають помилки з портами (наприклад, 5060, 3000 або 9093 зайняті), знайди процеси, які їх займають, і запитай у користувача дозвіл на їх завершення.
